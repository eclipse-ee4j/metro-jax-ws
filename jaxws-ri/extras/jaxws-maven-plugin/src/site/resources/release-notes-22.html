Release Notes - jaxws-maven-plugin - Version 2.2

Bug

  • [JAX_WS_COMMONS-39] - add encoding customization support for jaxws-maven-plugin
  • [JAX_WS_COMMONS-40] - wsgen not always accept dependencies
  • [JAX_WS_COMMONS-45] - Make it possible to use JAXB 2 plugins
  • [JAX_WS_COMMONS-46] - jaxws-maven-plugin >= 1.10 does not handle xjcArg correctly for XJC extnsion plugins
  • [JAX_WS_COMMONS-52] - JAX-WS 2.2 support
  • [JAX_WS_COMMONS-53] - External dependency of type "pom" is not working
  • [JAX_WS_COMMONS-71] - On windows, the generated files are using ANSI encoding
  • [JAX_WS_COMMONS-82] - jaxws maven plugin should fail build when encountering errors
  • [JAX_WS_COMMONS-84] - Show how to attach binding files to Metro with Maven

Improvement

  • [JAX_WS_COMMONS-3] - wsgen maven task should support attaching WSDL to artifacts
  • [JAX_WS_COMMONS-4] - wsimport maven task should support compilation from maven repository
  • [JAX_WS_COMMONS-35] - maven-jaxws-plugin should allow dependent libraries in provided scope
  • [JAX_WS_COMMONS-80] - Support for WSDLs in dependencies
  • [JAX_WS_COMMONS-83] - Expose new wsgen/wsimport options introduced in RI 2.2.6 through the plugin

New Feature

  • [JAX_WS_COMMONS-43] - Allow SEI classes to be discovered by introspection
  • [JAX_WS_COMMONS-69] - Support for -inlineSchemas
